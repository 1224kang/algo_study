## 병합 정렬
> **분할 정복** 방식을 사용해 데이터를 분할하고 분할한 집합을 정렬하며 합치는 알고리즘 
- 시간복잡도 : ```O(nlogn)```

❓ ```분할 정복``` 알고리즘
= 해결할 수 없는 문제를 작은 문제로 분할하여 문제를 해결하는 방법
(ex.) 퀵 정렬, 합병 정렬, 이진탐색, 선택 문제, 고속 푸리에 변환..

**⭐2개의 그룹을 병합하는 과정**
- **투 포인터 개념**을 사용 -> 왼쪽, 오른쪽 그룹 병합
- 왼쪽 포인터와 오른쪽 포인터의 값을 비교 -> 작은 값을 결과 배열에 추가 & 포인터를 오른쪽으로 1칸 이동

### 1517 : 버블 정렬 프로그램2
✏️ 범위가 500,000인데 1초 안에 실행이 완료되어야 하므로 시간복잡도가 ```O(nlogn)``` 인 **병합 정렬**을 수행해야 한다.

❓swap이 몇 번 일어났는지 알 수 있는 방법<br/>
=> 기존의 배열에서 특정 값이 **앞으로 몇칸 이동했는지 = swap 횟수**<br/>
예를 들어, 5가 앞으로 4칸 이동한 자리의 인덱스로 정렬되었따면, 5는 swap을 4번 실행한 것이다

∴ 정렬 과정에서 index가 이동한 거리를 result에 저장한다
